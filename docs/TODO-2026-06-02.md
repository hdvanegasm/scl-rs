# scl-rs Audit TODO — 2026-06-02

Derived from `scl-rs-audit-2026-06-02.md`. Ordered by severity. Do not treat
any item as done until covered by a test.

## HIGH

- [ ] **#1 Constant-time arithmetic.** Replace NAF/branching `scalar_mul`
  (`src/math/ec/secp256k1.rs:199`) with a constant-time ladder; remove the
  early-exit in `to_naf` (`src/math/field/secp256k1_scalar.rs:16`). Swap
  `random_mod_vartime` and `invert_mod` (vartime) for constant-time variants on
  secret-derived values. Replace the `Mersenne61` `From` subtraction loop
  (`src/math/field/mersenne61.rs:21`) with constant-time reduction. If
  constant-time is out of scope, document the non-CT status loudly.

- [ ] **#2 On-curve validation.** Add a checked constructor / custom
  `Deserialize` for `Secp256k1` that enforces `y^2 == x^3 + 7`
  (`src/math/ec/secp256k1.rs:14`). Call it on every received point. Validate
  Feldman commitments before the exponentiation in `is_valid`
  (`src/ss/feldman.rs:29`). Reject the identity where the protocol forbids it.

- [ ] **#3 Authenticated identities.** Switch TLS to mutual auth: server
  `with_client_cert_verifier`, client presents a cert
  (`src/net/mod.rs:292`). Derive `PartyId` from the authenticated certificate
  instead of the self-declared integer read at `src/net/channel.rs:149`. Reject
  identity/slot mismatches.

## MEDIUM

- [ ] **#4 Bind & deduplicate Feldman commitments.** Verify all shares against a
  single canonical dealer commitment vector, not each share's own copy
  (`src/ss/feldman.rs:79`). Reject disagreeing per-share vectors. Bind the
  commitment to dealer id + session id + threshold + participant set. Consider a
  commit-before-reveal/broadcast step for full VSS.

- [ ] **#5 Validate party indices.** In Shamir and Feldman `secret_from_shares`
  / `shares_from_secret` (`src/ss/shamir.rs:64`, `src/ss/feldman.rs:68`), reject
  indices that are zero in the field or not pairwise distinct, after reduction
  into the scalar domain, before interpolation. Do not rely solely on
  `all_different` in `compute_lagrange_basis` (`src/math/poly.rs:107`).

- [ ] **#6 Strengthen Feldman verification.** Reject identity coefficient
  commitments; verify the constant-term commitment (`commitments[0]`) against the
  agreed group public key; apply the length check to the canonical dealer
  commitment (`src/ss/feldman.rs:30`).

## LOW

- [ ] **#7 Fix Mersenne61 sampling bias.** Change `while value > modulus` to
  `>= modulus` in `random` and `random_non_zero`
  (`src/math/field/mersenne61.rs:43,52`); drop the now-redundant reduction or use
  a vetted uniform sampler.

- [ ] **#8 Reject empty polynomials on deserialize.** Add a `try_from` /
  custom `Deserialize` for `Polynomial` enforcing non-empty coefficients, or
  length-check before `evaluate`/`degree` (`src/math/poly.rs:31,35,49`).
